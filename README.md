# Завдання 3 (goit-algo-hw-05) Порівняння алгоритмів пошуку підрядка

## Опис завдання
Було реалізовано три класичних алгоритми пошуку підрядка:
- Боєра-Мура
- Кнута-Морріса-Пратта (КМП)
- Рабіна-Карпа

Для кожного алгоритму проведено тестування на двох текстових файлах.

## Використані тексти
1. **text1.txt** — “Використання алгоритмів у бібліотеках мов програмування”
2. **text2.txt** — “Методи та структури даних для реалізації бази даних рекомендаційної системи соціальної мережі”

## Підрядки
- Існуючий: `"алгоритм"`
- Вигаданий: `"abrakadabra"`

## Замірювання часу
Використано модуль `timeit`. Кожен алгоритм запускався 10 разів для обох підрядків у кожному тексті.

## Приклад результатів (після виконання)

|-----------|------------------|-------------------|------------------|-------------------|
| Boyer-Moore | 0.0013 c | 0.0019 c | 0.0022 c | 0.0029 c |
| KMP | 0.0018 c | 0.0023 c | 0.0021 c | 0.0030 c |
| Rabin-Karp | 0.0037 c | 0.0041 c | 0.0040 c | 0.0047 c |

(приблизні значення)

## Висновок
- Алгоритм **Боєра-Мура** показав найкращий час роботи в обох текстах.
- Алгоритм **КМП** стабільний, але трохи повільніший.
- Алгоритм **Рабіна-Карпа** найповільніший через обчислення хешів.

Отже, для великих текстів Боєр-Мур найефективніший, а КМП є хорошим універсальним варіантом.
